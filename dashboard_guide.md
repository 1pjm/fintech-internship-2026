# 전세대출 추천 대시보드 화면 설명

> `jeonse_scoring_pipeline.ipynb` Cell 0~7 실행 후 생성된 `41_jeonse_recommendation.csv`를
> Streamlit이 읽어 대시보드를 구성합니다.

---

## 화면 구성 순서 (위 → 아래)

```
┌──────────────────────────────────────────────────────┐
│  타이틀 + 파일명 캡션                                  │
├──────────────────────────────────────────────────────┤
│  ① 요약 지표 (KPI 카드 5개)                           │
├───────────────────────┬──────────────────────────────┤
│  ② 액션 분포 바차트    │  ③ 추천 점수 분포 바차트      │
├───────────────────────┼──────────────────────────────┤
│  ④ 상품 유형별 추천    │  ⑤ 고객 유형 분포 바차트     │
│     건수 바차트        │                              │
├──────────────────────────────────────────────────────┤
│  ⑥ 가장 많이 추천된 상품 Top 10 바차트                │
├──────────────────────────────────────────────────────┤
│  ⑦ 고객별 추천 결과 — 필터 + 데이터 테이블            │
├──────────────────────────────────────────────────────┤
│  ⑧ 고객 1명 상세 보기                                 │
└──────────────────────────────────────────────────────┘
```

---

## ① 요약 지표 (KPI 카드 5개)

대시보드 최상단에 숫자 카드 5개가 가로로 나열됩니다.

| 카드 | 표시 내용 | 설명 |
|------|----------|------|
| **전체 고객** | 500명 | CSV에 있는 고유 user_id 수. 추천 가능/불가 전체 포함 |
| **추천 고객** | ~257명 | Gate를 1개 이상 통과해 TOP5 추천을 받은 고객 수 |
| **추천 불가** | ~243명 | 모든 상품에서 Gate 탈락한 고객 수 |
| **메인 추천(건)** | ~725건 | action = "메인 추천"(점수 70점 이상)인 추천 행 수 |
| **추천 평균점수** | ~63.0점 | 추천 고객 기준 score 평균값 |

> **발표 포인트**: 500명 중 절반 정도만 추천 가능한 이유는 Gate 조건(무주택·세대주·신용점수 등)이 엄격하기 때문. 243명의 탈락 사유는 아래 테이블에서 확인 가능.

---

## ② 액션 분포 바차트

Gate 통과 여부와 상관없이 **전체 행(추천 불가 포함)** 의 action 값 분포를 보여줍니다.

| action 값 | 점수 기준 | 의미 |
|-----------|----------|------|
| 메인 추천 | 70점 이상 | 상단에 강하게 노출할 상품 |
| 리스트 노출 | 55~69점 | 추천 목록에 표시 |
| 조건 확인 유도 | 40~54점 | "조건 맞추면 가능" 안내 필요 |
| 추천 보류 | 40점 미만 | 후보이나 우선순위 낮음 |
| 추천 불가 | Gate 탈락 | 자격 미달로 추천 제외 |

> **발표 포인트**: "메인 추천"이 얼마나 되는지 보면 스코어링이 얼마나 잘 분별하고 있는지 알 수 있음. 추천 불가가 많은 것은 실제 전세대출 자격 요건이 까다롭다는 것을 반영.

---

## ③ 추천 점수 분포 바차트

**Gate 통과(추천 가능) 고객의 score**를 10점 단위로 구간화해 히스토그램처럼 보여줍니다.

- X축: 점수 구간 (0, 10, 20 … 100)
- Y축: 해당 구간에 속한 추천 행 수

> **발표 포인트**: 점수가 한쪽으로 몰리지 않고 넓게 분산되면 스코어링이 상품을 잘 변별하고 있다는 신호. 만약 90~100 구간에 대부분 몰린다면 정규화 로직을 재검토 필요.

---

## ④ 상품 유형(source)별 추천 건수 바차트

추천 고객에게 실제로 노출된 상품이 **어느 유형에서 왔는지** 보여줍니다.

| source 값 | 내용 |
|-----------|------|
| 정부지원 | 버팀목 A1~A5 (HUG/HF 보증, 저금리) |
| 은행 | 금감원 API 기반 시중은행·인터넷은행 상품 |
| 저축은행 | 금감원 API 기반 저축은행 상품 |
| 보험 | 금감원 API 기반 보험사 전세자금대출 |

> **발표 포인트**: 정부지원 상품(A1~A5)은 자격 조건이 맞는 고객에게 강제 노출(forced=1) 처리가 되어 있어, 청년·신혼·신생아 고객에게는 반드시 1개 이상 포함됨. 은행 상품이 대부분을 차지하더라도 정부지원이 적절히 섞이면 분산이 잘 된 것.

---

## ⑤ 고객 유형(profile) 분포 바차트

500명의 고객이 **어떤 유형으로 분류되었는지** 보여줍니다 (고객당 1개 유형, 중복 없음).

| 유형 | 판별 조건 |
|------|----------|
| 신용취약형 | 신용점수 600~699점 |
| 상환위험형 | 예상 DSR ≥ 35% |
| 한도부족형 | 신청금 > 상품 최대한도 90% |
| 신혼·신생아형 | 혼인 7년 이내 or 출산 2년 이내 |
| 청년무주택형 | 만 19~34세 + 무주택 세대주 |
| 금리민감형 | 부부합산 연소득 ≤ 4,000만원 |
| 기본형 | 위 유형 모두 해당 없음 |

> **발표 포인트**: 여러 유형에 동시 해당하면 "실패 비용이 큰 보호형" 우선 배정(신용취약 > 상환위험 > 한도부족 > …). 유형 분포를 보면 샘플 고객 데이터의 특성을 파악할 수 있음.

---

## ⑥ 가장 많이 추천된 상품 Top 10 바차트

**추천 결과에서 가장 많이 등장한 상품명 10개**를 내림차순으로 보여줍니다.

> **발표 포인트**: 특정 상품이 압도적으로 많이 추천된다면 쏠림 현상. "동일 금융기관 최대 2개" 정책(company dedup)을 적용했기 때문에 한 은행의 상품이 TOP5를 독점하는 것을 방지함. 최다 추천 상품 비중이 19% 정도면 분산이 잘 된 것 (분산 전에는 한 상품이 90%+ 차지했음).

## ⑦ 고객별 추천 결과 — 필터 + 데이터 테이블

### 필터 (상단 4개 컨트롤 + 검색)

| 컨트롤 | 역할 |
|--------|------|
| **최소 점수** (슬라이더) | 설정값 미만 score의 행을 숨김 (0~100) |
| **액션** (멀티셀렉트) | 보고 싶은 action만 선택 (메인 추천만 보기 등) |
| **고객유형** (멀티셀렉트) | 특정 profile 유형 고객만 필터링 |
| **상품유형** (멀티셀렉트) | 정부지원·은행 등 source별 필터링 |
| **상품명/사유 검색** (텍스트) | 상품명 또는 추천 사유에 키워드 포함된 행만 표시 |

### 데이터 테이블

필터 조건을 모두 만족하는 행을 user_id → score 내림차순으로 정렬해 표시합니다.

| 컬럼 | 내용 |
|------|------|
| `user_id` | 고객 ID |
| `profile` | 고객 유형 (금리민감형, 신용취약형 등) |
| `matched_types` | 해당된 전체 유형 목록 (콤마 구분) |
| `product_name` | 추천 상품명 |
| `source` | 상품 유형 (정부지원 / 은행 / 저축은행 / 보험) |
| `guarantee_agency` | 보증기관 (HUG / HF / SGI) |
| `rate_%` | 대표 금리 (최저·최고 평균, %) |
| `limit_만원` | 상품 최대 대출 한도 (만원) |
| `score` | 최종 추천 점수 (0~100) |
| `action` | 추천 강도 (메인 추천 / 리스트 노출 / 조건 확인 유도 / 추천 보류 / 추천 불가) |
| `reason` | 추천 또는 제외 사유 (최대 3개 근거) |

### 하단 요소

- **행 수 캡션**: "표시 중: N행 / 전체 M행 · 고객 K명" — 필터 결과 규모 확인
- **CSV 다운로드 버튼**: 현재 필터 결과만 UTF-8 BOM CSV로 내려받기

> **발표 포인트**: 추천 불가 고객(action = "추천 불가")도 reason 컬럼에 탈락 사유가 기록됨. 예) "무주택 아님—기금 불가 / 세대주 아님—기금 불가". 실제 서비스라면 이 사유를 고객에게 안내해 자격 개선 방향을 제시할 수 있음.

## ⑧ 고객 1명 상세 보기

화면 최하단에 위치한 **개별 고객 드릴다운** 기능입니다.

### user_id 드롭다운
500명의 user_id 목록에서 1명을 선택하면 해당 고객 정보가 즉시 갱신됩니다.

### 프로파일 표시
```
프로파일: 금리민감형  ·  매칭: 금리민감형,신혼·신생아형,기본형
```
- **프로파일**: 최종 배정된 유형 1개 (가장 높은 우선순위)
- **매칭**: 해당 고객이 조건을 충족하는 전체 유형 목록

### 추천 상품 테이블
선택한 고객에게 추천된 최대 5개 상품이 score 내림차순으로 표시됩니다.

| 컬럼 | 내용 |
|------|------|
| `product_name` | 상품명 |
| `source` | 상품 유형 |
| `guarantee_agency` | 보증기관 |
| `rate_%` | 대표 금리 |
| `limit_만원` | 대출 한도 |
| `score` | 추천 점수 |
| `action` | 추천 강도 |
| `reason` | 추천 사유 |

> **발표 포인트**: 특정 고객을 골라서 "이 고객은 왜 이 상품이 1위인가"를 설명할 수 있음. 예를 들어 청년 고객을 선택하면 버팀목 청년전용(A2)이 강제 포함되어 있고, reason에 "정부지원 저금리(연 2.8%) 자격 충족"이 표시되는 것을 보여줄 수 있음.

## 발표 흐름 제안

1. **대시보드 전체 스크롤** — 어떤 요소들이 있는지 한 번 훑어보기
2. **① KPI 카드** — "500명 중 257명이 추천 가능, 평균 점수 63점"
3. **② 액션 분포** — "메인 추천 725건, 스코어링이 잘 분별하고 있다"
4. **⑤ 고객 유형 분포** — "7개 유형으로 분류, 각자 다른 가중치 적용"
5. **⑥ Top 10 상품** — "최다 비중 19%, 쏠림 없이 분산됨"
6. **⑦ 필터 시연** — 예) 고객유형 = 신용취약형으로 필터 → 해당 고객 추천 결과 확인
7. **⑧ 상세 보기** — 특정 고객 1명 선택 → 추천 사유 설명
